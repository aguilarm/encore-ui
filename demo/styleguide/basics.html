Introduction

  • The goal of this sub-site is to get development teams at Rackspace not familiar with the Encore product a set of design patterns so all parts of Encore look like one finished, unified product.
  • This section of the Demo site focuses mainly on markup, CSS and design patterns.
  • The rxApp common.less file defines many base CSS rules and classes for app use. Included in this is normalize.css. This helps create a consistent starting point for styles across all browsers.
  • If you're curious about specific Encore-specific angular.js directives, find the "All Components" section in the left hand navigation.
  • If you're a Rackspace team outside of Encore and need help setting up an new project from an engineering perspective go to this Getting Started guide in the Encore-UI repository.
  • Kevin Lamping wrote a document to address coding standards while writing CSS.

Typography

  • Encore specific: We use Google Fonts, in particular the Roboto font. Google Fonts is not included in the Encore-UI package at this time, so you will need to add this to your index.html file.
  • The Encore-UI default font is Roboto. This is used for all text on the page and is loaded via Google Fonts. Read the README.md file for rxApp on fonts for more information.

Heading/Title Styles

  • Headings are sized using 'xl', 'lg', 'sm' and 'xs'.
  • The size of the heading is not impacted by the heading level used (e.g. <h1>, <h2>). This is done to ensure that the heading levels are sequential.
  • Encore specific: For design consistency, we usually use the default medium size (<h2 class="title">) for most subheaders. Other headers classes can be used for context.

The .subdued class is a basic color change to make text less prominant. Add hover style if a link or inside a link.
Full Width
Class that marks content as full width (e.g. to give a button full width). Requires 'display' to be defined as inline-block or block.
Hidden content
  • .hidden class will add to hide an element but not remove it from the spacing of the page.
  • .visually-hidden - Used to hide content visually but let them still be read by screen readers (removed from the page spacing)
Clearing floats
  • Clearfix - Provide clearfix functionality See http://nicolasgallagher.com/micro-clearfix-hack/
