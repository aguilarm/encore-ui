Heading/Title Styles

Headings are sized using 'xl', 'lg', 'sm' and 'xs'.

The size of the heading is not impacted by the heading level used (e.g. <h1>, <h2>). This is done to ensure that the heading levels are sequential. See https://github.com/rackerlabs/encore-ui/blob/master/guides/css-styleguide.md#basic-rules for further reasoning.

For design consistency, we usually use the default size (<h2 class="title">) for most subheaders. Other headers classes can be used for context. You can also add the subdued class - this is the class used as a subtitle in rxApp/rxPage.

Descriptions / Metadata

The Encore-UI metadata component is a list of terms with their associated descriptions. This is used throughout Encore to show metadata, especially on an individual object such as a server or a CBS volume. This is an ideal solution if you have anywhere from 5-10 definitions; anything more than that - or anything with that displays visual or image content, and you may want to consider a different design pattern. (In Encore, see what we use in Support Services for an example of this.)

Lists

By default, we do not list styles for lists. Add bullets/decimals to lists with the 'list' class. It is also possible to build non-semantic lists using the additional item class.

Use this in conjunction with icons, colors and helper classes such as .flush-right to build out action menus. Look at the detail pages in Cloud Servers and view-source for additional context.
