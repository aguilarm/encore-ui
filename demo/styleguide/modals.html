Design Patterns within Encore

  • In Encore, when do we use modals versus requiring a whole new page? What is the threshold for what we want to do in a modal?
    • According to Bootstrap, "Modals are streamlined, but flexible, dialog prompts with the minimum required functionality and smart defaults."
    • For Encore, we use modals for confirmation messaging. For notification messages we use rxNotification, the static subdirective of rxNotify. See the example below, replacing type="error" with the appropriate message response.
    • HEADS UP: We also have used modals to edit content in lieu of having an edit view. This design pattern is not ideal and an Edit View will need to be considered as a pattern in future iterations.
  • Using the rxModalForm subdirective will automatically provide you with a title, subtitle, submit and cancel text. You may use rxForm elements within rxModalForm.
  • For real world examples, take a look at the ng-template blocks triggered from server action links under Encore Cloud.
  • If you need to deviate from the title / footer pattern at all, you'll need to take a look at the rxModalForm directive template markup and build upon that.
