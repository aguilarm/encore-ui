Basic Usage

  • Basic usage of the rxModalAction angular.js component. Using rxModalForm which will automatically comes with a title, subtitle, submit and cancel text.
  • Address notifications (static)

Design Patterns within Encore

  • All this comes with design patterns we already use for the rest of Encore so you don't have to worry about design, but if you need to deviate from that at all, you'll need to take a look at how the rxModalForm directive was built and build upon that.
  • When do we use modals versus requiring a whole new page? What is the threshold for what we want to do in a modal?
  • Strategy for communicating errors / validation messages from the server - should there be direction on how we do this?

UI Roadmap / Possible Future-work

  • We should document patterns of how modals are titled and what button text is (see LBaaS for examples--there are some inconsistencies here Encore-wide)
